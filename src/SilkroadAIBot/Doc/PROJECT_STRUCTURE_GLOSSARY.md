# Silkroad AI Bot - Project Structure & File Glossary

This document provides a complete directory of the `src/SilkroadAIBot` codebase, describing the responsibility of each functional component and the files within them.

## 1. Directory Overview
| Component | Purpose |
| :--- | :--- |
| **Bot/** | High-level AI, state management, and combat logic. |
| **Networking/** | Protocol parsing, opcode registry, and SecurityAPI integration. |

## 2. Component File Glossary

### 🤖 Bot (AI & Orchestration)
Files in `src/SilkroadAIBot/Bot/`
*   **BotController.cs**: The main operational loop; coordinates the update cycles of all logic "Bundles".
*   **WorldState.cs**: The centralized "Source of Truth" for the game world (Nearby entities, current target, character stats).
*   **SkillController.cs**: Manages skill cooldowns and casting requirements.
*   **TargetingSystem.cs**: Implements selection logic (e.g., nearest monster, aggressors first).
*   **TrainingArea.cs**: Defines the operational radius and script-based bounding boxes for the AI.
*   **AI/AIBrain.cs**: The high-level decision engine that picks the next action for the bot to perform.
*   **Agents/SocialAgent.cs**: Orchestrates "human-like" behaviors and LLM-powered chat integrations.
*   **Bundles/**:
    *   **AttackBundle.cs**: Implements combat sequences and buff management.
    *   **RecoveryBundle.cs**: Automates HP/MP and Status Recovery (Pills).
    *   **LootBundle.cs**: Handles item filtering and pickup logic.

### 🔌 Networking (Protocols)
Files in `src/SilkroadAIBot/Networking/`
*   **PacketHandler.cs**: The primary dispatcher for both Client-to-Server and Server-to-Client traffic.
*   **PacketParser.cs**: The main decoder for game-state packets (Inventory, Skill Lists, Guilds).
*   **PacketSender.cs**: The official encoder for outgoing requests (Moving, Casting, Selecting).
*   **WorldStateAnalyzer.cs**: Specialized parser that feeds Spawn/Despawn and coordinate data into `WorldState`.
*   **LoginManager.cs**: Manages the sequence of Gateway Handshakes, Login, and Character Selection.
*   **ClientlessConnection.cs**: High-performance TCP wrapper with integrated SecurityAPI encryption.
*   **SecurityAPI/**: (Folder) Contains Blowfish, CRC, and Silkroad-specific handshake implementations.

### 💾 Data (Asset Management)
Files in `src/SilkroadAIBot/Data/`
*   **DataManager.cs**: The primary game data engine. It extracts monster/item/skill metadata from Joymax PK2 files.
*   **DatabaseManager.cs**: A thread-safe SQLite wrapper for persisting bot settings and caching game encyclopedia data.
*   **NavmeshReader.cs**: Parsers for `Map.pk2` navigation cells used for real-time 3D pathfinding.

### 🛰️ Proxy (The MITM Hook)
Files in `src/SilkroadAIBot/Proxy/`
*   **ProxyManager.cs**: Manages the lifecycle of the local proxy server that intercepts `sro_client.exe`.
*   **ProxyContext.cs**: Maintains state for an active game session, linking the Client and Server TCP streams.
*   **ProxyListener.cs**: The actual socket listener that accepts the redirected client connection.

### 🧱 Models (Data Entities)
Files in `src/SilkroadAIBot/Models/`
*   **SRCharacter.cs**: Extended state for the user's own character (EXP, Gold, Stat points).
*   **SRModel.cs**: The base definition for all 3D objects (Monsters, NPCs, Players).
*   **SRSkill.cs/LearnedSkill.cs**: The distinction between Skill Metadata (Cooldowns) and Player State (Level).
*   **SRCoord.cs**: 3D Coordinate system with Silkroad-specific "Region" encoding.

### 🖥️ UI (User Workspace)
Files in `src/SilkroadAIBot/UI/`
*   **MainForm.cs**: The application's main entry window and dashboard logic.
*   **ThemeColors.cs**: Registry for the custom Silkroad-inspired dark theme palette.
*   **Controls/PacketSnifferControl.cs**: An integrated real-time packet debugger for developers.

### ⚙️ Core (Shared Logic)
Files in `src/SilkroadAIBot/Core/`
*   **Opcodes.cs**: A comprehensive lookup table for Silkroad Protocol Opcodes.
*   **Helpers/BotLogger.cs**: A high-speed, thread-safe logging framework with multi-output support.
*   **Configuration/ConfigManager.cs**: JSON-based persistent setting management.
