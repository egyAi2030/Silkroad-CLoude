## 2. Stability & Infrastructure Fixes

### 2.1 Zombie-Proof IPC Bridge
The original "Pipe instances are busy" error was caused by lingering background processes holding onto the same pipe name. 
- **Fix:** Implemented a **Dynamic Pipe Name** system (`SilkroadAIBot_IPC_{PID}`).
- **Resilience:** Increased max pipe instances to 10 and opened security policies to allow cross-integrity connections (Standard Client -> Admin Bot).
- **C++ Sync:** Updated `RedirectorDLL` (dllmain.cpp) to dynamically scan for and find the active bot's unique pipe ID.

### 2.2 Connection (C8) & Login Resolution
The "C8" error was traced to a mismatch between the gateway port (32390) and the internal proxy listener (31411).
- **Protocol Fix:** Modified `SroLoader.cs` to correctly write the `proxy_config.ini` with the distinct `proxyPort`.
- **Human Mimicry:** Added a mandatory **9-second delay** in `LoginManager.cs` after the server list is pulled, preventing the "too fast" disconnections common on private servers like NOVA.
- **Auto-Selection:** Configured the bot to automatically select the first character slot and enter the world if no specific name is provided.

### 2.3 Administrator & UAC Support
The bot was re-enabled with its original `app.manifest`. It now correctly requests **Administrator privileges** on startup.
- **Requirement:** This is essential for the `SroLoader` to perform DLL injection into `sro_client.exe`.
- **Execution:** Provided the `RunAs` PowerShell command to trigger the elevation prompt from the CLI.
